Application framework

This pattern outlines some of the qualities that make a application consistent, familiar, and usable.

Overview

In order to facilitate the user experience, we have created a generic app layout that helps make the app consistent and usable.

the architecture of this layout is mainly to make the most of the screen width for this reason the navigation is made up of icons

Anatomy

1. Header name: For Werfen product name. This should always and only link the user to the domainâ€™s home page. In many cases the baseline is used to give context to the user about de product (eg: laboratory name)
2. Utilities (optional): The header provides a home for global system-level utilities from anywhere in the site. Utility icons should not be used to directly navigate to an area. Instead, they should open a panel that provides access to other places in the product.
3. Logged user: Provide information about the logged user.
4. Main menu (optional): The main menu icon is used to open product navigation, in a big applications you could use a second navigation bottom aligned.
The main menu icon used must identify the application module it represents

Behaviours
