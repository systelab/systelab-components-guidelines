Login

The login page allows a user to gain access to an application by entering their user ID and password.

Anatomy

1. Subtitle: For Werfen product name. (In some cases a product had many applications)
2. Title: For Werfen application name.
3. Version information: Show the version and bundle information.
4. Logo: Is the product logo, check your product team's guidance and choose an asset accordingly.
5. Required fields: The user ID and password fields are both required although.
6. Remember user: Saves the user ID and presents the completed input field the next time the user logs in.
7. Submit button: Button label should be 'Log in'.
8. Forgot username/password link: Takes users to a page where they can recover their username and/or password.
9. Brand logo: Show Werfen logo and copyright related information.
10. Background image: Check your product team's guidance and choose an asset accordingly.

Errors and validation

Client-side errors

Validate as much of the user's data before submission as possible. This real-time validation should happen when the input field loses focus and checks for input errors like invalid characters and empty fields. This helps users easily identify mistakes and fix them before submitting the login form.

Server-side errors

If there are server-side errors when the user submits the login form, the page should be reloaded, the password field cleared, and the user returned to the username input field. Use an inline notification to display the errors and provide clear direction on how users should resolve the issue.
