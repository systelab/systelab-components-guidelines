Toast

Toasts display brief, temporary notifications. They're meant to be noticed without disrupting a user's experience or requiring an action to be taken.

Overview

Toasts are non-modal, time-based window elements used to display short messages. They usually appear at the bottom of the screen and disappear after a few seconds.

Toast or Message popup?

Toasts should only be used for confirmations, simple notifications, and low-priority alerts that do not need to completely interrupt the user experience.
Message popus are ideal for when a situation requires a user's attention, either for displaying important information or prompting for a response.

Placement

Toast position differs depending on platform scale. By default, a toast is placed at the bottom end for desktop scale and at the bottom center for mobile scale, to avoid disrupting the user experience.

Title
  • The title should be short and descriptive, explaining the most important piece of information.
  • When possible, communicate the main message using just the title.
  • Limit content to one sentence.
Body
  • Be concise and avoid repeating or paraphrasing the title
  • Limit content to two sentences.
Action
  • Keep labels concise and clearly indicate the action the user can take.
  • Limit action labels to one or two words.
Don't include a redundant action
Don't display more than one action

Sizing

Toast notifications use a max width depends on the placement. Limit toast notifications to one line bt title and one line by body.

Behaviors

Auto-dismissible

By default, a toast will dismiss when the user clicks the close button. A toast also has the option to auto-dismiss. Be sure to set a minimum of 5 seconds so that users can have time to read the toast message. If an actionable toast is set to auto-dismiss, make sure that the action is still available elsewhere in the app.

Toast with automatically dismiss and action button, it is important that there are alternative routes to navigate to the link destination.

Close button

When the notification is important, the user can be forced to dismiss it manually via the close button.
