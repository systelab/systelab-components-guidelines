Context Menu

Use the context menu component when additional options are available to the user but there is a space constraint

Placement

Use the context menu when the user needs to perform actions on a particular element of the UI. Some very clear use cases:

  • Data grid: we will use it to perform actions on a row such as deleting a record
  • Cards: let's suppose we have some cards that represent instruments of a system, in the context menu you could put the actions on each instrument as for example: stop, rename, delete, restart, etc

Usage

Text

The text within an overflow menu should be direct so users can quickly decide on an action. Actions that could cause a significant change to the userâ€™s data (delete app, delete service, etc.) are separated by a divider and live below the primary set of actions.

Nested

Nested menus are an extension of dropdown menus. They save screen space by organizing a long list of items into categories that the user can click to reveal deeper levels.

Scroll

Scroll is available in cases where there are a lot of menus.

For a better user experience avoid scroll or nested menus.
