Message popup

Focus the user’s attention exclusively on one task or piece of information via a window.

Overview

  • Message popup are a variant of dialog used to present critical information or request user input needed to complete a user’s workflow.
  • Message popup interrupt a user’s workflow by design.
  • When active, a user is blocked from the on-page content and cannot return to their previous workflow until the modal task is completed or the user dismisses the modal.

Per defecte els message popup utilitzen la mida petita del dialog

When to use

  • Notify the user of urgent information:
    Use a modal dialog to notify the user of urgent information concerning their current work. Modal dialogs are commonly used to report system errors or convey a consequence of a user’s action.
  • Confirm a user decision:
    Use a modal dialog to confirm user decisions. Clearly describe the action being confirmed and explain any potential consequences that it may cause. Both the title and the button should reflect the action that will occur. If the action is destructive or irreversible then use a transactional danger modal.
Message popup require the user to take an action to dismiss them.
