Message popup

Focus the user’s attention exclusively on one task or piece of information via a window.

Overview

  • Message popup are a variant of dialog used to present critical information or request user input needed to complete a user’s workflow.
  • Message popup interrupt a user’s workflow by design.
  • When active, a user is blocked from the on-page content and cannot return to their previous workflow until the modal task is completed or the user dismisses the modal.
  • Message popup use a small size of dialog

When to use

  • Notify the user of urgent information:
    Use a modal dialog to notify the user of urgent information concerning their current work. Modal dialogs are commonly used to report system errors or convey a consequence of a user’s action.
  • Confirm a user decision:
    Use a modal dialog to confirm user decisions. Clearly describe the action being confirmed and explain any potential consequences that it may cause. Both the title and the button should reflect the action that will occur. If the action is destructive or irreversible then use a transactional danger modal.
Message popup require the user to take an action to dismiss them.

A warning message vs. an error message

An error message alerts people of a problem that has already occurred. By contrast, a warning message alerts people of a condition that might cause a problem in the future.

Variants

Variant Purpose
Warning Warning modals appear in anticipation of a significant change. Have empathy for people. Inform, but don't alarm. If the warning comes before an action, clearly communicate what will happen if they proceed, and provide an alternative or opt-out where possible.

Use a warning section message to help users:

  • Avoid errors
  • Manage authentication issues
  • Take the actions needed to avoid potentially dangerous actions
  • Feel certain they're making the decision, for example, in confirmation modals
Error System anomalies, something didn’t work

Use an error section message to let the user know when:

  • Something destructive or critical has happened
  • Access has been denied
  • there are connectivity issues
Information

The information section message is the default appearance used to signify a change in state or important information.

Question Confirmations (potentially destructive actions that need user confirmation to proceed)
