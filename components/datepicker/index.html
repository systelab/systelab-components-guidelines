Datepicker

Pickers are used to display past, present, or future dates.

Anatomy

1. Label: Instructs the user what to do with the control.
2. Date field: A text input field where the user can manually type in the date.
3. Icon: fa-calendar-day icon indicates the calendar menu is available.
4. Calendar: The menu where a date may be selected.
5. Month and year label: Indicate the current year and month frame.
6. Previous and next month/year controls: Allows the user to move forward or backward time.
7. Day: Days in the month

Design guidelines

  • Date picker is a calendar-view popover. It is paired with a text input so that it can send a formatted date value to the input.
  • Date picker is shown by clicking, or gaining focus on the input field.
  • Today’s date is selected with border blue. Unless preloaded with another date, today’s date will be selected in the Date Picker.
  • Clicking on the check-calendar icon inside the Date Picker returns the focus to today’s date.
  • Focus is indicated by a grey background.

Usage

A Date picker is used to select a date when a user needs to understand the date in context with other possible dates. Date choices are influenced by day of the week, or week within a month.

Date pickers aren’t helpful when the date is a known value like a birthdate. A user will likely want to type that value in versus hunting for it in a Date Picker. In cases where space is limited, it may be a good idea to omit the Date Picker.

A date picker is also helpful when the formatting of the date is important.
