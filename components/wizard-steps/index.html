Wizard steps

The wizard steps is a visual representation of a userâ€™s progress through a set of steps, guiding toward the completion of a specified process.

Overview

When to use it

  • Use wizard steps to keep the user on track when completing a specific task
  • When the global goal could be divided in sub-tasks
By dividing the end goal into smaller, sub-tasks, it increases the percentage of completeness as each task is completed.

When not to use

  • Do not use the wizard steps as the primary navigation menu.
  • When the are only one step.
  • When the are a lot of sub-tasks.

Anatomy

1. Step button: Indicates the step number and it is used to navigate through the steps. Also indicates if the step it's done or it's pending.
2. Title step: Help the user to context de step
3. Summary (optional): Any extra or aditional help to the user.
4. Separator: Line to join steps, when the step it's done the line is filled blue to indicate the finished step.
Display the steps in order from left to right. Indicate to the user that they are performing a multistep process, and show the direction of movement.

Placement

The wizard steps must be align centered in the container

Behaviors

  • Clicking in the button step, allow the user to return to a previous step to review their data submission.
  • For the pending steps the buttons are disabled.

Variant

Vertical stepper

Vertical steppers are designed for narrow screen sizes or mobile devices.
